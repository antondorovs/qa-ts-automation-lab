# Bug Reports

This folder stores bug report examples for QA practice and portfolio review.

Use these files to practice writing clear, reproducible defects with enough context for a developer, QA engineer, or product owner to understand the problem.

## Good Bug Report Checklist

- The title describes the user-visible problem.
- Steps are numbered and reproducible.
- Expected and actual results are separate.
- Environment is specific enough to reproduce the issue.
- Evidence includes screenshots, logs, videos, API responses, or test output when available.
